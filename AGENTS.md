## What Torpedo is (and is not)

Torpedo is a **fork of [rayfish](https://github.com/rayfish/rayfish)**, a P2P mesh VPN powered by [iroh](https://iroh.computer). It is not original software; almost all architecture, protocol design, and code are upstream's. The fork exists for **one product goal**: make the overlay IPv4 subnet **configurable** so Torpedo can run alongside Tailscale on the same host. Upstream hardcodes `100.64.0.0/10` (the CGNAT range Tailscale also claims) and refuses to start next to Tailscale; Torpedo defaults to `10.88.0.0/16` and lets the operator pick any CIDR per network.

The binary is **`torpedo`**. The Cargo **package/library is still named `rayfish`** (`[package] name = "rayfish"`, `[[bin]] name = "torpedo"`), so all internal `use rayfish::…` paths, the `info,rayfish=debug` log filter, and the crate's own module names are deliberately unchanged — renaming the crate would be churn with no user-visible benefit and is out of scope.

## KEEP-ON-PURPOSE — do NOT rename these

The fork deliberately keeps a set of `rayfish` identifiers. These are load-bearing (they must match upstream infrastructure or would be pointless churn) and are protected by constraints in `spec/design_spec.py` + `reconcile.py`. **An agent must not "finish the rename" on any of these:**

- **Crate / library name `rayfish`** and every `use rayfish::…` path, plus the `info,rayfish=debug` tracing filter. Internal only; never user-visible.
- **Relay / discovery presets** — the `"rayfish"` config keyword and its preset URLs (`relay.iroh.rayfish.xyz`, `dns.iroh.rayfish.xyz`). Protected by **CON-001**; `reconcile.py`'s `relay_preset_untouched` check requires the literal `"rayfish" => Ok(preset.to_string())` to remain in `src/config.rs`. Renaming would point nodes at nonexistent infrastructure.
- **Author attribution** (`dario@rayfish.xyz` in `Cargo.toml`) — upstream authorship is retained honestly.
- **The reserved resolver IP `magic_dns_v4` (e.g. `10.88.100.53`)** — Magic DNS itself was removed (MINIMAL-012), but `membership::magic_dns_v4(subnet)` and the reservation that keeps this subnet-derived address off the member IP pool are retained for **wire compat (D1)**: a full-torpedo node on a shared network routes it to its own resolver, so it must never be assigned to a member. (The `.ray` TLD naming scheme is otherwise gone with Magic DNS.)

Everything that *was* renamed (binary, ALPNs, listen port, config/log/socket paths, host DNS artifacts, deep-link scheme, launchd label, config-dir env var, DHT record names) is a **collision-prone host artifact or user-facing surface** that had to change so torpedo and rayfish can coexist on one machine. **CON-007** (`reconcile.py`'s `host_identity` check) is a curated-token regression gate that keeps those from creeping back in; it scans `src/**/*.rs` for a specific token list and must report `leak_count == 0`.

## Spec-first workflow (libspec + reconcile.py)

Changes in this fork are **spec-driven and committed one requirement at a time**:

- **`spec/design_spec.py`** — one Python class per requirement/constraint. The ID is the first line of the docstring (e.g. `SUBNET-001`, `RENAME-006`, `CON-007`). `Requirement` subclasses are structural/design (verified by reading the diff + code); `Constraint` subclasses carry an `enforcement_logic` Jinja expression and are the automatable gates. A `UserStory` captures intent.
- **`reconcile.py`** — the per-commit gate. Run `python3 reconcile.py` from the repo root; it must exit `0`. It runs eleven checks and prints a JSON context: `build` (cargo build), `clippy` (`-D warnings`, 0 warnings), `test` (cargo test), `grep_hardcoded_cgnat` (no stray `100.64`/`100.100.100` literals in `membership.rs`/`tun.rs`/`dns.rs` beyond allowed comment lines), `relay_preset_untouched` (**CON-001**, must equal `"rayfish"`), `host_identity` (**CON-007**, curated-token `leak_count` must be `0` across `src/**/*.rs`), `build_tooling_identity` (**CON-008**/RENAME-010, the same curated-token approach for non-Rust build tooling — `justfile`/`contrib/` — that CON-007's `src/**/*.rs` scan can't cover; `unexpected_count` must be `0`), `report_identity` (**CON-009**/RENAME-014, curated `rayfish` product-name tokens on the diagnostic/report + repo surface — `src/**/*.rs` plus `.github/**` and `cliff.toml`; `unexpected_count` must be `0`), `cli_reference_identity` (**CON-010**/RENAME-016..017, a regex — `(?<![.\w-])ray (?=[a-z])` — that keeps the pre-fork `ray <verb>` binary reference out of `src/**/*.rs` **and** the `tests/` harness while the lookbehind skips the KEEP forms `.ray`/`ray-proto`/`rayfish`; `unexpected_count` must be `0`), and `test_harness_identity` (**CON-011**/RENAME-017, curated functional `rayfish` service/config/marker/record tokens in `tests/` — `systemctl … rayfish`, `/etc/rayfish`, `Added by rayfish`, `_rayfish_certgen`, `rayfish/files/1` — kept out of the e2e harness while the `NAMES=(rayfish-*)` cloud-VM labels stay; `unexpected_count` must be `0`), and `test_subnet_identity` (**CON-012**/SUBNET-015, keeps the pre-fork `100.64` CGNAT range and `100.100.100` magic-DNS IP out of `tests/` — the fork defaults to `10.88.0.0/16` with the resolver at `10.88.100.53`, so a stale literal is functional breakage; `unexpected_count` must be `0`).
- **libspec** — spec snapshots are recorded **manually after each commit**: `/home/erik/code/libspec/.venv/bin/libspec link --vcs git --revision <sha> --metadata "..."`. `libspec diff` previews. There is **no `build` command**. Each such link mutates `.libspec/libspec.jsonl`, which is folded into the next commit.

**The loop for a change:** amend/add the requirement class in `spec/design_spec.py` → implement → `python3 reconcile.py` until green → commit (conventional subject; **no authorship trailers of any kind**) → `libspec link` the new sha.

# Product & architecture reference

P2P mesh VPN powered by [iroh](https://iroh.computer). Connects peers by cryptographic identity (EndpointId), not IP address. Dual-stack addressing: stable IPv4 in a **configurable** overlay subnet (default `10.88.0.0/16`, an uncommon 10.x slice that avoids Tailscale's `100.64.0.0/10`; FNV-1a of identity, host-bit width follows the prefix) and stable IPv6 in `200::/7` (blake3 of identity, 120-bit, never rotates).

## Build

```bash
cargo -q build                 # add --features tor for Tor transport
cargo -q check
cargo -q test
cargo -q clippy
cargo bench                    # Criterion microbenchmarks of the per-packet data path (benches/forward.rs)
cargo build --release          # distributable binary at target/release/torpedo
```

The crate splits into a library (`src/lib.rs`, daemon modules as `pub mod`) and a thin binary (`src/main.rs`, the `torpedo` CLI/IPC client, `use rayfish::…` — the library name is kept, see KEEP-ON-PURPOSE). The split lets benchmarks (`benches/`) and integration tests reach the internal data path; `cargo install` builds the binary against the in-package library unchanged.

**justfile:** `just cross`/`just deploy`/`just deploy-dev` identity was fixed (`binary := "torpedo"`, `groupadd torpedo`, `systemctl restart torpedo`) — safe to use. `ray-mobile` was removed in MINIMAL-016 (no Android build in tetron).

## Run

The daemon (`torpedo daemon`) owns the TUN device and iroh endpoint and runs as a system service. CLI commands talk to it over Unix-socket IPC.

```bash
sudo torpedo up                # install+start the service, then activate the VPN
torpedo create [--name n] [--hostname h] [--subnet CIDR] [--tor]   # always closed (approval-gated); MINIMAL-013 removed --open. --subnet overrides the default 10.88.0.0/16 for this network. Prints room id (public key)
torpedo join <room-id-or-invite> [--name alias] [--hostname h] [--tor]  # join by room id (queues for approval) or, on a full-torpedo network, an invite code
torpedo leave <net> | nuke <net>   # nuke = publish empty record then leave
torpedo kick <net> <peer>      # coordinator-only (closed networks): remove a member. Prunes it from the roster + approved list, republishes the signed blob, disconnects it mesh-wide. Refused on open networks (target would re-join) and against coordinators/self
torpedo status                 # all networks (works without daemon); per-host traffic, member count excludes self. Ends with a `pending` summary of things awaiting the user (join requests) each with the command that clears it
torpedo <cmd> --json           # global flag: machine-readable JSON for status/requests/admin list (color + spinners off)
torpedo up [--hostname h] | down   # activate / standby. down takes the data plane (TUN) offline but stays connected to peers (still online); --hostname sets your default name

torpedo requests <net>         # coordinator-only: peers awaiting live approval
torpedo accept <net> <id> | deny <net> <id>    # admit / reject a pending join request
torpedo admin <net> add <id> | list            # coordinator-only: grant the network key (co-coordinator) / list key-holders. <id> is the target's short id as shown in `torpedo status` (its endpoint-id prefix) — unlike `--peer` elsewhere, hostname is NOT accepted and resolves with "could not resolve identity"
torpedo config [get [key] | set <key> <value> [--replace] | unset <key>]   # global daemon settings; keys: relay, discovery-dns, subnet. relay/discovery values are comma lists of presets (rayfish/n0), URLs, or IPv4s (default augments n0; --replace swaps them out; n0/empty resets). `subnet` takes a single CIDR (e.g. 10.88.0.0/16; empty resets to default). Written client-side to settings.toml; all apply on `sudo torpedo restart`
torpedo completions <shell>
torpedo version | torpedo --version | torpedo -V        # print the compiled version + git sha
```

> **Self-update is REMOVED (MINIMAL-002).** tetron deletes the entire `update` / `auto-update` machinery (upstream torpedo only neutralized it behind the now-retired CON-006). There is no `torpedo update`, no `auto-update`, no `--auto-update` flag, and no `src/update.rs`. Upgrade by replacing the binary and running `sudo torpedo restart`.

> **File sharing and device pairing are REMOVED (MINIMAL-004).** tetron deletes `torpedo send`/`files`, `torpedo pair`/`unpair`, the `FILES_ALPN`/`PAIR_ALPN` accept arms, `onepassword.rs`, `revocation.rs`, the `_torpedo_certgen` cert-generation floor, and `DeviceUserMap`. The identity model collapses to **one device = one user**. Copy files with `scp`/`rsync` over the mesh IPs; back up the identity key yourself (`<config_dir>/secret_key` is one `0600` file). **D1 wire compat is preserved:** `control.rs` keeps the `DeviceCert`/`PairMsg`/`CertRefresh`/`Unpaired` types so a full-torpedo peer on a shared network is decoded-and-ignored, never errored.

> **The declarative apply layer and node-local aliases are REMOVED (MINIMAL-011).** tetron deletes `torpedo apply` (+ `--example`/`--dry-run`/`--prune`/`--invite-missing`), `torpedo alias`, `torpedo identityof`, `src/apply.rs`, `src/cli/alias.rs`, `src/daemon/mesh/alias.rs`, the `Alias*` IPC ops, and the per-network `aliases` config/status field. Reconcile a fleet with a script over `torpedo status --json`. This landed before MINIMAL-010 because `apply`/`identityof` code lived in `src/cli/firewall.rs` and consumed the firewall-suggest IPC; removing them first kept every commit compiling and coherent.

> **Admission is approval-only; invite minting is REMOVED (MINIMAL-013).** `torpedo create` always makes a closed (`Restricted`) network — `--open`/`--closed` are gone. The only way onto a tetron-coordinated network is **live approval**: a joiner dials the room id, lands in the pending queue, and a coordinator runs `torpedo accept` (`torpedo requests`/`deny` round it out). tetron deletes the whole `torpedo invite` create/list/revoke CLI + `InviteAction`, the single-use `InviteStore` ledger + its `invites/<network>.toml`, the `InviteCreate`/`InviteList`/`InviteRevoke` IPC ops (+ `InviteCreated`/`InviteListResponse`/`InviteInfo`), the `invite_create`/`reusable_key_create`/`invite_list`/`invite_revoke` daemon handlers, reusable-key **minting**, the `InviteShare`/`InviteUsed` gossip **senders** (`gossip_to_coordinators`/`gossip_targets`/`sender_is_coordinator`), and the per-network `invite_lock` mutex. **Kept:** joiner-side invite redemption (`invite::encode/decode_invite_code`, `JoinRequest.invite_secret` — a tetron node can still *join* a full-torpedo network by invite code); blob reusable-key **validation** on admission (`membership::validate_reusable_key`, the only invite a tetron coordinator honors); `requests`/`accept`/`deny`; `admin add`/`list` (co-coordinator grant is the availability story for admission); and `kick`. **D1 wire compat:** `GroupMode::Open` stays understood (a tetron node granted admin on a full-torpedo open network still auto-admits per the signed blob — only *creating* open networks is gone); `InviteShare`/`InviteUsed` from a full co-coordinator are decoded and ignored, never errored. `membership.rs` is left textually untouched (its `ReusableKey::from_secret`/`revoke_reusable` helpers stay close to torpedo for cherry-picks).

> **Hostname is fixed at join; rename propagation and ephemeral auto-kick are REMOVED (MINIMAL-014).** tetron deletes `torpedo hostname` and `torpedo ephemeral`, the `SetHostname`/`SetEphemeral`/`GetEphemeral`(+`EphemeralStatus`) IPC ops, `MeshManager::set_hostname`/`announce_rename_to_peers`/`set_ephemeral`/`get_ephemeral`, `src/daemon/mesh/rename.rs` (the `pending_hostname` drain + `rename_satisfied`/`has_pending_hostname`), `spawn_stale_member_pruner`/`should_prune`, the `pending_hostname` and `ephemeral_ttl_secs` config fields, the status line's `ephemeral` TTL, and the reconverge worker's 30s rename-backstop tick (it is now purely trigger-driven). A member's hostname is set once at join; the coordinator still **resolves collisions at admission** (`admit_peer` → `resolve_collision`, suffix `-1`, `-2`, …), and a member adopts that authoritative name from the signed roster on reconverge (the trimmed `reconcile_local_hostname`). The coordinator control reader no longer acts on a `MeshHello` hostname (it still captures a full-torpedo peer's `device_cert` off it — D1). `outgoing_hostname` survives (moved into `join.rs`) to announce the fixed name on reconnect. **Manual `kick` remains** the remediation tool for stale members.

> **Magic DNS and all OS DNS mutation are REMOVED (MINIMAL-012).** tetron deletes `src/dns.rs`, `src/dns_config.rs`, `src/dns_resolver.rs`, `src/dns_packet.rs`, `src/daemon/dns_manager.rs`, the port-53 intercept in `forward.rs`, the `magic-dns`/`dns-upstreams` config keys, the `DnsManager` satellite + `HostnameTable`/`ReverseLookupTable`, the panic-hook resolv.conf restore, the `zbus`/`inotify`/`simple-dns` (and macOS `system-configuration`/`core-foundation`) deps, and the `tests/e2e/dns` scenario. **Reach peers by mesh IP from `torpedo status`** (`--json` for scripts); host naming is `/etc/hosts`' job. Hostnames still ride the signed roster and show in `torpedo status`; `kick <hostname>` resolves against the roster, not DNS. The one surviving piece is `membership::magic_dns_v4` — the `.100.53` address stays reserved (never assigned) for D1 wire compat (see KEEP-ON-PURPOSE). The daemon's host footprint shrinks to: TUN device, routes, config dir, log dir, unix socket.

> **The userspace firewall is REMOVED (MINIMAL-010).** tetron deletes `torpedo firewall` (all subcommands), `src/firewall.rs`, `src/cli/firewall.rs`, `src/daemon/mesh/firewall.rs`, `src/reject.rs`, `src/picker.rs`, `firewall.toml`, the `auto_accept_firewall` config key + `--auto-accept-firewall` join flag, all firewall IPC ops/types, and the firewall e2e scenario. **Packet filtering is now the host firewall's job — every mesh peer reaches every port a local service binds** (mesh membership still gates *who* can connect; a network is a reachability boundary). Restrict ports with nftables/ufw on the TUN interface (e.g. `nft add rule inet filter input iifname "torpedo" tcp dport != 22 drop`). `forward.rs` keeps only the upstream ingress anti-spoof check (a peer may only source packets from its own mesh IP); `src/packet.rs` holds the IP-header parser the forwarder still needs (peer routing, anti-spoof, the port-53 Magic-DNS intercept). **D1 wire compat is preserved:** the `GroupBlob.suggested_firewall` field and `ray-proto`'s `policy.rs`/`firewall.rs` wire types stay — a coordinator's suggestions are carried through the blob verbatim on republish but never acted on.

**Privilege & access (Tailscale operator model):** the always-root daemon does privileged work; clients are unprivileged. The IPC socket is mode `0666`; authority comes from a per-request `SO_PEERCRED` UID check in `MeshManager::check_authorized()`, not socket permissions. Reads (`status`, `*… show`) are open to any local user; mutating commands need root or the configured `operator_uid`; `set-operator` is root-only. Only `install`, `restart`, `uninstall`, `start`, `stop`, `set-operator`, and `daemon` need `sudo`; everything else (incl. `up`/`down`) is IPC. `torpedo up`/`install` auto-grant operator to `$SUDO_USER`.

```bash
sudo torpedo install | restart | uninstall   # manage the service unit/plist
sudo torpedo start | stop                    # start / stop the service. stop = fully offline (closes peer connections); start = back online
sudo torpedo set-operator <user>             # authorize a user to run torpedo without sudo
```

## Architecture

```
App → TUN (10.88.x.x / 200::x) → torpedo → iroh QUIC datagrams → peer
```

One iroh Endpoint and TUN device are shared across all networks. Each network gets its own ALPN (`torpedo/net/<version>/<pubkey-prefix>`); the `ProtocolRouter` dispatches incoming connections by ALPN to per-network handlers. The leading `<version>` (`transport::MESH_PROTOCOL_VERSION`) makes the ALPN the **mesh protocol-version gate**: iroh negotiates the ALPN during the QUIC handshake, so peers on different mesh versions share no common ALPN and cannot connect — no in-band version handshake exists. Bumping the constant on a breaking mesh change severs old peers automatically.

### Modules

- `src/main.rs` — thin clap CLI + IPC client: the `Cli`/`Command` definitions, `main` dispatch, tracing/panic plumbing, and shared presentation helpers. `torpedo daemon` (hidden) runs the foreground daemon loop. `build.rs` stamps the git short SHA shown by `torpedo version`. The command handlers are split into `src/cli/` — one file per domain (`network`, `status`, `invite`, `admin`, `service` — incl. `cmd_up`/`install_and_start_service`/`cmd_install`/`cmd_restart`/`cmd_uninstall_service`/`cmd_set_operator` and the shared process/daemon-reachability helpers). Each opens with `use crate::*;`; `src/cli/mod.rs` flattens them with `pub(crate) use <m>::*` (private `mod`s so only contents re-export — avoids clashing with the `use rayfish::{invite, …}` aliases), and `main.rs` does `use cli::*`, so every handler resolves the others (and the shared helpers) by name. Handlers and the consts/enums they cross are `pub(crate)`.
- `src/daemon/` — daemon process, split into a directory module. The core type is **`MeshManager`**: it holds the endpoint, TUN, `PeerTable`, networks map, blob store, and `ProtocolRouter`. (Magic DNS + its `DnsManager` satellite were removed by MINIMAL-012; hostnames now live only in the roster.) `mod.rs` holds the core type definitions (`MeshManager`/`NetworkState`/`NetworkHandle`/`MeshCtx`), the `impl MeshManager` IPC dispatcher (`handle_request`, `check_authorized`, `set_operator`, `mesh_ctx`, `register_coordinator_handler`/`promote_to_coordinator`, `resolve_short_id_any_network`), the roster-projection + control-message helpers, and unit tests. The bulk lives in `src/daemon/mesh/` — each file opens with `use super::super::*;` and is re-exported (`pub(crate) use mesh::*`) so callers reach the moved free functions by bare name. Per-domain IPC ops are `impl MeshManager` method-modules: `create_join`, `runtime`, `diagnostics`, `invite`, `admin`. Mesh machinery: `accept.rs` (`CoordinatorAcceptState`/`MemberAcceptState`/`AcceptHandler`/`MeshProtocol`/`ProtocolRouter`; closed-network pending-join queue bounded by `MAX_PENDING_JOINS`, oldest-eviction via `evict_oldest_pending`), `join.rs` (`join_mesh_shared` + `JoinParams` + `spawn_reconnect_loop`), `bootstrap.rs` (`run_daemon`, `build_daemon`, `serve_ipc`/`handle_ipc_client`). The former `background.rs` is five files: `publish.rs` (DHT network/contact/lazy publishers + `update_snapshot_and_publish`), `reconverge.rs` (`fetch_verified_blob`/`reconverge_and_apply`, group poller, `reconcile_local_hostname` — adopts the roster's authoritative name), `coordinator.rs` (coordinator control reader, peer cleanup, `remove_member_roster_only`/`finalize_removal` used by `kick`), `select.rs` (pure decision helpers — `coordinator_dial_order`, `pick_first_welcome`, `choose_path_index`, `persisted_roster`). (`rename.rs` was removed by MINIMAL-014; `outgoing_hostname` moved into `join.rs`.) `create_join.rs`'s `join_network_inner` reads as three phases — `resolve_and_fetch_blob` → `dial_fresh_join`/`dial_reconnect` → `finalize_join` — with per-join inputs bundled in `JoinContext`. **Shared-handle bundling:** `MeshCtx` groups the daemon-wide cheap-`Clone` handles (identity, `PeerTable`, `tun_tx`, stats, blob store, pruned-peers set) into one value built by `MeshManager::mesh_ctx()`, embedded in the accept states and passed as one argument to spawn/reconverge free functions. `MeshCtx::forward_ctx(disconnect_tx, token)` builds a peer's `forward::ForwardCtx`. Admission gate: `CoordinatorAcceptState::handle_connection` → `admit_peer` (open-mode auto-admit / valid reusable key / pre-approved via `torpedo accept`) vs queue-as-pending (closed, awaiting approval).  **Coordinator setup is shared:** `create_network_inner` and `restore_coordinator_network` both go through `seal_and_publish` + `spawn_coordinator_background_tasks`.
- `src/ipc.rs` — thin shim; the `IpcMessage` enum, `MsgpackCodec` (length-prefixed msgpack over Unix socket), and socket helpers live in `ray-proto`. Socket at `/var/run/torpedo/torpedo.sock`.
- `src/identity.rs` — persistent Ed25519 keypair (`<config_dir>/secret_key`, `0600`); device certs (`create/store/load_device_cert`). Resolves its dir via `config::config_dir()`.
- `src/invite.rs` — joiner-side invite-code encoding only (MINIMAL-013 removed the coordinator `InviteStore` ledger + minting). `encode_invite_code`/`decode_invite_code` = `bs58(network_pubkey(32) || coordinator(32) || secret(16))`; `torpedo join <code>` decodes it, dials the pinned coordinator, and presents the secret in its `JoinRequest`, so a tetron node can still redeem an invite minted by a full-torpedo coordinator. **Reusable keys live in the signed blob** (`membership::ReusableKey`, validated on admission via `validate_reusable_key`).
- `src/membership.rs` — `IdentityProvider`, IPv4/IPv6 derivation, `MemberList`/`ApprovedList`, `GroupBlob { members, approved, suggested_firewall, name, reusable_keys }` with canonical msgpack + blake3 hashing. **Subnet is configurable:** `default_subnet()` returns `10.88.0.0/16` (an uncommon 10.x slice avoiding Tailscale's `100.64.0.0/10`, SUBNET-011); the authoritative subnet rides the `GroupBlob`/config, host-bit width follows the prefix, and IP derivation is subnet-relative. `assign_ip` picks the lowest free collision index at admission; `validate_member`/`validate_approved`/`validate_no_duplicate_ips`/`resolve_ip_tiebreak` keep rosters consistent. `SuggestedFirewall`/`HostSuggestions` live in `ray-proto` (`policy.rs`).
- `src/transport.rs` — iroh endpoint setup, per-network ALPN (`network_alpn` → `torpedo/net/<MESH_PROTOCOL_VERSION>/<prefix>`); optional Tor transport (`tor` feature). The shared endpoint binds a **fixed UDP port** `TORPEDO_LISTEN_PORT` (**43737**), stable across restarts and manually port-forwardable for guaranteed direct reachability (iroh still does NAT traversal/UPnP/PCP, discovery, and relay fallback). If the port is in use the daemon warns and falls back to an ephemeral port. **Custom relay/discovery overrides:** `bind_endpoint` keeps `Endpoint::builder(presets::N0)` and conditionally overrides from `config::ServerOverride` — `build_relay_mode` swaps relays, `apply_discovery` stacks pkarr publisher/resolver per configured discovery URL. No-ops when unset. Set via `torpedo config set relay|discovery-dns ...`.
- `src/tun.rs` — async dual-stack TUN (IPv4 + IPv6 /128), split into `TunReader`/`TunWriter`. `configure_ipv6()` assigns the TUN's own IPv6 at creation. `route_peer_range()` installs the `200::/7` peer-range route **after link-up** (from `MeshManager::activate()` post-`set_link_up`). `route_self_loopback(v4, v6)` installs `-host` routes for our own addresses via `lo0` (macOS only) so self-traffic is answered locally. Idempotent across `up`/`down`.
- `src/forward.rs` — TUN ↔ peer forwarding via dual-stack routing lookup; labeled drop counters. `spawn_peer_reader` takes its data-plane handles as a single `ForwardCtx` bundle. Inbound datagrams pass only the upstream ingress anti-spoof check (a peer may only source packets from its own mesh IP); packet *filtering* is the host firewall's job (MINIMAL-010). (Magic DNS and its port-53 intercept were removed by MINIMAL-012; the embedded mesh SSH server and its port-22 NAT by MINIMAL-003 — remote shells are the host sshd's job over the mesh IPs.)
- `src/packet.rs` — minimal IP-header parser (`PacketInfo`/`parse_packet_info`) for the data path: peer routing, the ingress anti-spoof check, and the port-53 Magic-DNS intercept. Extracted from the removed firewall module (MINIMAL-010); it is not firewall logic.
- `src/dht.rs` — one pkarr record per network (blob hash + seed peers + `m,<mesh-version>`); only the coordinator (per-network secret key) can publish. The pkarr relay URL defaults to `dns.iroh.link/pkarr` but follows the `discovery-dns` config when set.
- `src/control.rs` — length-prefixed msgpack control protocol over QUIC streams (`JoinRequest`, `JoinPending`, Welcome, MemberApproved, MeshHello, BlobUpdated, `AdminGrant`, `InviteShare`, `InviteUsed`, `Ping`/`Pong`, …). **D1 wire compat:** the `DeviceCert`/`PairMsg`/`CertRefresh`/`Unpaired` types and `JoinRequest.device_cert`/`MeshHello.device_cert` fields are **retained but inert** — a full-torpedo peer still sends them over the mesh ALPN, so a tetron node must decode-and-ignore (never error), and always sends `device_cert: None`. All triggers pass through the `ControlGate` rate limit.
- `src/peers.rs` — `PeerTable` (dual v4/v6 DashMaps). A peer keeps one virtual IP across every network it joins, so each `PeerEntry` holds a *set* of connections; `lookup_v4/v6` return a `PeerRoute`. `remove_peer_from_network()` drops one network's route, `remove()` drops it everywhere.
- `src/config.rs` — config storage. **Sharded, atomic, per-network:** globals in `settings.toml` (`operator_uid`, `default_hostname`, the `ServerOverride` settings `relay`/`discovery_dns`, and the overlay `subnet`), each network in `networks/<name>.toml` (per-network secret/public key, `my_hostname`, `group_mode`, `admins`, `direct`, per-network subnet). Writes go through `write_file` (temp + atomic `rename`) and are targeted. `load()` assembles the in-memory `AppConfig`, running a one-time `migrate_legacy`. **Location:** `config_dir()` is `/etc/torpedo` on Linux (system service), `~/.config/torpedo` on macOS; dirs `0750 root:torpedo`, secret-bearing files `0600 root:root`, non-secret `0640 root:torpedo` (the `torpedo` group is created on Linux install). **`ServerOverride` + subnet** carry pure resolver/validator helpers; `config_set`/`config_get` back the `torpedo config` CLI (subnet takes a single CIDR; n0/empty resets server keys; unknown key/bad URL/IP/CIDR rejected before persist). **The relay preset keyword `"rayfish"` is retained (CON-001) — do not rename.**
- `src/hostname.rs` / `src/network_name.rs` — hostname + local-alias generation and collision resolution (`resolve_collision` appends `-1`, `-2`, …).
- `src/stats.rs` — iroh-metrics `ForwardMetrics` counters for the forwarding data path (rx/tx/drop reasons) + the periodic stats logger. The Prometheus `:9090` exporter, `PeerMetrics`, and `torpedo report` were removed by MINIMAL-009.
- **CLI presentation**: plain text output (no colors, spinners, or glyphs). `--json` on every read command provides machine-readable output. The `style.rs`, `layout.rs`, and `progress.rs` modules were removed in MINIMAL-015.
- `src/logdir.rs` — daemon log directory (`/var/log/torpedo` on Linux, `/Library/Logs/torpedo` on macOS). Rolling daily files via `tracing-appender` (7 most recent retained).
- `src/ratelimit.rs` — `ControlGate`: per-connection token-bucket guard + strike counter over inbound control messages. `check()` returns `Verdict::Allow`/`Drop`/`Close`. One per control-listener task.
- `src/shutdown.rs` — SIGINT/SIGTERM via `CancellationToken`.

### Key flows

- **Create:** generate per-network `SecretKey` → derive addresses from the network's subnet (default `10.88.0.0/16`, or `--subnet`) → build initial `GroupBlob` → publish blob + signed pkarr record → persist keys + `group_mode` + subnet → print public key as the room id. Always closed (`Restricted`) — MINIMAL-013 removed `--open`.
- **Access modes & admission:** the room id (network public key) is a published discovery key, **never** an admission credential. A tetron network is always **closed**: an unknown peer is queued in `NetworkState.pending`, surfaced via `torpedo requests`, and admitted with `torpedo accept` (**live approval** — the only admission path tetron offers). A tetron coordinator additionally honors a **reusable key** whose hash rides the signed `GroupBlob.reusable_keys` (validated, never minted here). **Open**-mode auto-admit and the `GroupBlob.reusable_keys` set are still understood for **D1 wire compat** — a tetron node granted admin on a full-torpedo network coordinates it under that network's mode. The handler is `CoordinatorAcceptState`, run by **any node holding the network key**; the admitting coordinator assigns the joiner's IPv4 via `assign_ip`.
- **Join handshake:** resolve pkarr record → fetch + verify `GroupBlob` → dial in `coordinator_dial_order` (invite-pinned coordinator first, then other `is_coordinator` members) until one replies `Welcome` → send `JoinRequest { invite_secret? }` first → coordinator replies `Welcome`, `JoinPending`, or `JoinDenied`. A bare room id on a closed net yields `JoinPending` (queued for `torpedo accept`); `invite_secret` is set only when joining a full-torpedo network by invite code or reusable key. Then connect to other members with `MeshHello` and poll pkarr for blob updates.
- **Gatekeeper:** any coordinator (any network-key holder) can approve identities and broadcast `MemberApproved`; admitting a fresh joiner survives any single coordinator being offline. The coordinator need not be online for *member* reconnects at all.
- **DHT (single-record):** one pkarr record per network signed by the per-network secret key. The pkarr address *is* the network public key, so records can't be spoofed. `spawn_group_poller()` refetches the blob every 60s when the hash changes.
- **Reachability model (segmentation-first):** a network is a reachability boundary — two peers exchange packets iff they share ≥1 network (a QUIC connection only exists within a shared network). Access control is the network split alone; there is no per-device packet filter (MINIMAL-010). Within a shared network every peer reaches every port a local service binds — restrict that with the host firewall (nftables/ufw) on the TUN interface. The only in-daemon ingress check is upstream anti-spoofing (a peer may only source packets from its own mesh IP).
- **Multiple admins = shared network key.** `torpedo admin <net> add <id>` grants the key to a member over the authenticated mesh ALPN (`AdminGrant`), making it a co-coordinator that can publish the signed blob and admit fresh joiners. The grantee calls `promote_to_coordinator` on receipt.
- **Hostname (fixed at join):** rename propagation was removed (MINIMAL-014) — there is no `torpedo hostname`. A member's hostname is set once from its join `JoinRequest`/`MeshHello`; the coordinator resolves collisions authoritatively at admission (`admit_peer` → `resolve_collision`, suffix `-1`, `-2`, …), and a member adopts that final name from the signed roster on reconverge (`reconcile_local_hostname`). Since invite minting is gone (MINIMAL-013), a tetron coordinator has no invite-bound authoritative hostname to enforce (the `admit_peer` authoritative path stays only for D1 — a full-torpedo invite binding a coordinator would honor).
- **Reconnection:** per-peer reader detects drop → coordinator removes the dead peer; joiner reconnects with exponential backoff (1s–30s) then re-sends `MeshHello`.
- **Control-plane abuse defense:** both control read loops gate each connection with a per-task token bucket (`ratelimit::ControlGate`). Over-budget messages are dropped; a sustained flood trips `Verdict::Close` (`forward::ABUSE_CODE`). `MemberSync`/`BlobUpdated` only `notify_one()` a per-network debounced reconverge worker (~300ms coalesce, single-in-flight).
- **Leave:** `torpedo leave` gracefully closes its connections with `forward::LEAVE_CODE`. Peers see `DisconnectEvent.intentional = true`; the coordinator prunes the member, republishes, and broadcasts a payload-free `MemberSync`. A plain timeout/reset is *not* intentional.
- **Kick:** `torpedo kick <net> <peer>` is coordinator-authoritative, closed-networks only. `kick_member` removes it from the roster + approved, drops DNS, republishes, broadcasts `MemberSync`, and closes its own link(s) with `KICK_CODE`. **Mesh-wide teardown is roster-driven:** `prune_departed_peers` closes + unroutes any live connection whose peer is no longer in the applied roster, recording it in `MeshCtx.pruned_peers` so the reconnect loop skips the re-dial.
- **up/down (data plane) vs start/stop (whole daemon):** the daemon connects every saved network at startup (control plane) and keeps those connections for its lifetime. `activate()`/`deactivate()` toggle only the **data plane** (TUN link, routes, inbound forward gate). `torpedo down` is standby (still online to peers, no traffic); `torpedo up` is near-instant. `sudo torpedo stop`/`start` is fully offline/online.
- **Tor (optional):** `--tor` adds `TorCustomTransport` alongside relay; onion address derived from the iroh `SecretKey`. Needs a Tor daemon (`ControlPort 9051`).

## Conventions

- Use `cargo -q` for all cargo commands; `tracing` for logging. `main::init_tracing` composes layers (console + file + optional OTLP) with **split filters**: console/CLI stays at `info`, rolling daily files under `logdir::log_dir()` capture our crate at `debug` (`info,rayfish=debug` — the crate name `rayfish` is kept, so the filter target is too; dependencies stay at `info`). `RUST_LOG` overrides both. Returns a `LogGuard` that must stay alive for the process.
- Tracing carries spans: network lifecycle handlers use `#[tracing::instrument]`; the per-peer reader + reconnect loop wrap tasks in `info_span!("peer"/"reconnect", …)` so the rolling-file logs are correlatable.
- Panics are fail-fast in the daemon: `main::install_panic_hook` (set only for `torpedo daemon`) records the panic, appends it to `panic.log`, then `std::process::abort()`. The service unit restarts it; `panic.log` persists in the log dir.
- Never share I/O resources (TUN, sockets, streams) behind a Mutex — split into read/write halves. Avoid Mutex generally: prefer channels, atomics, or `RwLock`/`ArcSwap`.
- CLI subcommands carry short `visible_alias`es (clap): `create`→`new`, `leave`→`rm`, `status`→`st`/`ls`, `version`→`ver`; action verbs `list`→`ls`, `remove`→`rm`/`del`, `show`→`ls`/`list`, `add`→`a`, `revoke`→`rm`, `approve`→`ok`. Aliases must be unique within each `#[derive(Subcommand)]` enum.
- ALPN per network: `torpedo/net/<version>/<pubkey-prefix>` (`MESH_PROTOCOL_VERSION` then first 16 hex chars). **The version segment in every ALPN is that protocol's compatibility gate.** Each protocol versions independently — bump the relevant version in the same change that breaks its wire format. iroh negotiates the ALPN at the QUIC handshake, so peers on different versions share no common ALPN and can't connect. `torpedo join` pre-checks the coordinator's signed `m,<mesh-version>` and bails with a precise message; `transport::connect_to_peer_with_alpn` maps an ALPN-mismatch connect error to an "incompatible version" hint.
- TUN MTU 1280 (IPv6 minimum link MTU, RFC 8200 §5; matches WireGuard/Tailscale). Wire format (control + IPC): 4-byte BE length + msgpack body.
- Room id = per-network public key string (discovery only). tetron networks are always closed: joining needs operator approval (`torpedo accept`), a reusable key, or (against a full-torpedo coordinator) an invite code. Invite code = `bs58(pubkey || coordinator || secret)`. Local aliases (adjective-noun-noun) are display-only.
- Config under `config::config_dir()` (`/etc/torpedo` on Linux, `~/.config/torpedo` on macOS): `secret_key`, `settings.toml`, `networks/<name>.toml` (one per network). MINIMAL-013 removed the coordinator invite ledger, so there is no longer an `invites/` subtree. Pre-migration installs auto-split the old `networks.toml`. On Linux the tree is `root:torpedo`; secret-bearing files are `0600 root:root`. CLI commands that write identity directly need root on Linux since the tree is under `/etc`.
- Keep commit subjects conventional (`feat`/`fix`/`docs`/`style`/`ci`/…) so git-cliff can render release notes. **Commit messages contain ONLY a description of what was done — never an authorship trailer (`Co-Authored-By`, `Author`, etc.) of any kind.**
- Always update docs (this `AGENTS.md`, `README.md`) after finishing a feature or significant change. And record the change in the spec-first workflow: amend/add the requirement in `spec/design_spec.py`, keep `reconcile.py` green, and `libspec link` the commit.
- Keep `CHANGELOG.md` current as part of every change (`## [Unreleased]`, Keep a Changelog format: `Added`/`Changed`/`Fixed`/`Performance`), describing behavior from the user's perspective. Skip pure-internal churn with no user-visible effect.
